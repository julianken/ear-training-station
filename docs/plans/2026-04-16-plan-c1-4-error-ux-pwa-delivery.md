# Plan C1.4 — Error UX + PWA + Delivery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close out Plan C1. Add the error/permission UX components (MicDeniedGate, DegradationBanner, ShellToast), wire the service worker via `@vite-pwa/sveltekit`, add the axe-core a11y smoke spec, document the bundle-size budget, and clean up the two small pre-existing deferred items (PR #10 `MicPermissionState.unavailable`, PR #47 Mergify `autoqueue`). After this plan merges, Plan C1 is done and the MVP ship criteria are all green.

**Architecture:** Error components are Svelte shell primitives that render based on shell-level stores (`degradationState`, `consecutiveNullCount`, `pendingToasts`). The `MicDeniedGate` renders in place of the session route when a permission-denied error fires during `startRound()`. Service worker is generated by `@vite-pwa/sveltekit` — precache the built shell + icons, runtime-cache the TF speech-commands model. Axe a11y smoke runs one Playwright spec that navigates each screen and asserts zero serious/critical violations.

**Tech Stack:** Existing SvelteKit 2.x setup + `@vite-pwa/sveltekit` + `@axe-core/playwright`.

**Spec:** `docs/specs/2026-04-16-plan-c1-ui-integration-design.md`

**Depends on:** Plans C1.1, C1.2, and C1.3 merged.

**TDD discipline — unit AND e2e:** continued. E2E tests are written before implementation for each user-visible error UX component. Service worker acceptance is a Playwright test that simulates offline and verifies the precached shell loads.

```svelte
<script lang="ts">
  import { pendingToasts } from './stores';
  import { onMount } from 'svelte';

  onMount(() => {
    const unsub = pendingToasts.subscribe((toasts) => {
      toasts.forEach((t) => {
        setTimeout(() => {
          pendingToasts.update((curr) => curr.filter((x) => x.id !== t.id));
        }, 3500);
      });
    });
    return unsub;
  });
</script>

<div class="toasts" aria-live="polite">
  {#each $pendingToasts as t (t.id)}
    <div class="toast toast-{t.kind}">{t.message}</div>
  {/each}
```

```typescript
import { seedOnboarded } from './helpers/app-state';

const screens = [
  { path: '/', label: 'station dashboard' },
  { path: '/settings', label: 'settings' },
  { path: '/scale-degree', label: 'exercise dashboard' },
];

for (const { path, label } of screens) {
  test(`axe: ${label} has no serious/critical a11y violations`, async ({ page }) => {
    await seedOnboarded(page);
    await page.goto(path);
    const results = await new AxeBuilder({ page }).analyze();
    const serious = results.violations.filter(
      (v) => v.impact === 'serious' || v.impact === 'critical',
    );
    expect(serious, JSON.stringify(serious, null, 2)).toEqual([]);
  });
}

test('axe: onboarding step 1', async ({ page, context }) => {
  await context.grantPermissions(['microphone']);
  await page.goto('/onboarding');
  const results = await new AxeBuilder({ page }).analyze();
  const serious = results.violations.filter(
    (v) => v.impact === 'serious' || v.impact === 'critical',
  );
  expect(serious, JSON.stringify(serious, null, 2)).toEqual([]);
});
```
